Algorithms

This page contains a very nice and detailed description of the algorithms used in this project, as well as amazing sliding comparisons of the original and filtered images.

CBIR - what do we battle?

Content bases information retrieval systems (CBIR) accept images as an input and a question similar to: what do you see on this picture? The systems returns a list of relevant images as an answer. That input image is represented in a vector that is based on the content of an image. You can consider this vector a numerical “fingerprint” of what is displayed on the image. A CBIR system then compares that vector to content based vectors of other images in a database and selects the most similar vectors as images that have something similar or something to do with the original image.

PIRE

Perturbations for Image Retrieval Error (PIRE) is an approach to battle automatic content retrieval from images by CBIRs. PIRE prevents the process of vector comparisons by perturbating the pixels of the original image. That way the vector of the original image is altered and does not resemble its actual “fingerprint” anymore. Similar vectors in the database are now similar to the “fake” fingerprint instead of the “real” fingerprint and show images that don’t have anything to do with the original picture. This results in a CBIR returning a list of images that are completely different from the original image. Since the perturbation happens on a pixel level, human eyes won’t recognise a perturbated image when they see one, but computers get confused and won’t be able to extract content anymore.

PERC

PerC perturbs images with respect to perceptual colour by minimizing the colour difference and maximizing the adversarial effect which is measured through misclassification. The higher the adversarial effect, the more unrecognisable an image becomes for a computer. The trick here is to make an image as unrecognisable for a computer as possible, without making it unrecognisable or “ugly” for the human eye. One of the main benefits of using PerC perturbations is that the changes in colour are less perceptible, especially in smooth regions of saturated colour. On top of that, PerC is able to hide large perturbations in the RGB space. The RGB space is the whole spectrum of colours described in proportions of red, green and blue. This game of hide and seek within the RGB space results in modifications of colours that are not readily noticeable for the human eye and alterations stay hidden for computersystems.
