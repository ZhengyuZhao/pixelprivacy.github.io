Algorithms

This page contains a short description of the algorithms used in this project, as well as amazing sliding comparisons of the original and filtered images. If you want to know the full mechanisms, go to the paper or repository using the buttons below each description. Please pull the slide to see the modified image.

PIRE

Perturbations for Image Retrieval Error (PIRE) is an approach to battle automatic content retrieval from images by CBIRs. PIRE prevents the process of vector comparisons by altering the pixels of the original image. That way the vector of the original image is changed and does not resemble its actual “fingerprint” anymore. Similar vectors in the database are now similar to the “fake” fingerprint instead of the “real” fingerprint and show images that don’t have anything to do with the original picture. This results in a CBIR returning a list of images that are completely different from the original image. Since the alteration happens on a pixel level, human eyes won’t recognise an altered image when they see one, but computers get confused and won’t be able to extract content anymore.

PerC

PerC alters images with respect to perceptual colour by minimizing the colour difference and maximizing the adversarial effect which is measured through misclassification. The higher the adversarial effect, the more unrecognisable an image becomes for a computer. The trick is to make an image as unrecognisable for a computer as possible, without making it unrecognisable or “ugly” for the human eye. This distance measurement was chosen through the results of a user study that showed how far apart colours could be before changes became noticed. One of the many benefits of using PerC is that the changes in colour are less perceptible, especially in smooth regions of saturated colour. On top of that, PerC is able to hide large perturbations in the RGB space. This game of hide and seek within the RGB space results in modifications of colours that stay hidden for computersystems. And the best part? You can use PerC in combination with other algorithms and filters to even make it more difficult for CBIRs to infer information from your picture.

Logit

In realistic scenarios, it is almost impossible to access the technical details of specific malicious information extraction algorithms. Logit is specifically designed to address this by also securing substantial protection against unknown algorithms. The merit of logit is that it does not need intensive resources that are necessary for other related solutions.
